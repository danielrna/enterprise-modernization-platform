# v0.5.2

External Quickstart Proof release for Phase 2 onboarding and consultant conversion.

## External quickstart proof

Make the first run Docker-first, explicit, and reproducible without reading project internals.

Audience: Java consultants and external repository maintainers trying EMP for the first time

- Adds a dedicated `docs/quickstart.html` page with Docker, GitHub Action, output paths, pack selection, and evidence interpretation.
- Rewrites the README first-run path around the published Docker image and expected `index.html` plus `report.json` outputs.
- Explains passed, failed, timeout, and not-applicable evidence so validation failures remain commercially useful instead of looking like tool failure.
- Links quickstart guidance from the homepage, generated documentation nav, Migration Hub, release notes, and Consultant Demo bundle.
- Preserves the Phase 1 proof base at 75 public reports, 75 checkout-backed reports, and 17 passing validation reports.

- [Quickstart](docs/quickstart.html)
- [Consultant demo](docs/consultant-demo.html)
- [Benchmark index](docs/benchmarks/index.html)

## Consultant conversion path

Connect the public proof set to a concrete client conversation: run, read evidence, explain limits, sell next work.

Audience: Consultants turning readiness evidence into paid validation and remediation work

- Adds quickstart assets to the downloadable Consultant Demo bundle.
- Positions honest validation failures as scope evidence for environment repair, module narrowing, or migration preparation.
- Keeps the no-backend story clear: static HTML and JSON reports are the deliverable.
