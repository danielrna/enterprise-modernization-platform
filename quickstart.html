Docker Path

Open the target repository. 
cd /path/to/spring-or-java-app

If the repository contains multiple samples, run from the actual application folder, not necessarily the repository root. Spring Guides-style repositories often need a subdirectory such as complete/.

Run the readiness analyzer. 
docker run --rm -v "$PWD:/workspace" danielrna/enterprise-modernization-platform:v0.5.4 analyze . --pack spring-boot-3-readiness --out reports/emp-readiness
Open the static report. 
reports/emp-readiness/index.html

What You Get

Machine-readable evidence for agents, CI, or later comparison.

Common First-Run Friction

Multi-sample repository

If the root has folders like initial/ and complete/, run EMP inside the app folder so build metadata and source scope match one application.

Already upgraded app

Spring Boot 3.x applications are not scored by the Spring Boot 2 to 3 pack. Use Jakarta, Java LTS, Spring Security, Hibernate, or JUnit readiness instead.

No score is still evidence

A non-applicable result means the tool avoided claiming a readiness score for the wrong migration path.

Use the JSON

report.json preserves the detected Spring Boot version, Java version, build tools, findings, and next actions for review.

GitHub Action Path

For an external repository, add this workflow and download the report artifact after the run.

name: EMP Readiness

on:
  workflow_dispatch:
  pull_request:

jobs:
  readiness:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - name: Run EMP readiness
        uses: danielrna/enterprise-modernization-platform@v0.5.4
        with:
          path: .
          pack: spring-boot-3-readiness
          out: emp-report

      - name: Upload EMP report
        uses: actions/upload-artifact@v4
        with:
          name: emp-readiness-report
          path: emp-report
