Run With Docker

docker run --rm -v "$PWD:/workspace" danielrna/enterprise-modernization-platform:v0.5.6 analyze . --pack jakarta-readiness --out reports/jakarta-readiness

Open reports/jakarta-readiness/index.html. The same folder contains report.json for automation, agents, and later comparison.

Run In GitHub Actions

name: EMP Readiness

on:
  workflow_dispatch:
  pull_request:

jobs:
  readiness:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - name: Run EMP readiness
        uses: danielrna/enterprise-modernization-platform@v0.5.6
        with:
          path: .
          pack: jakarta-readiness
          out: emp-report

      - name: Upload EMP report
        uses: actions/upload-artifact@v4
        with:
          name: emp-readiness-report
          path: emp-report

FAQ

Does this apply code changes?

No. The readiness command measures migration risk and writes static evidence. Use transform dry-run separately when you need change planning and validation evidence.

What files should I share?

Share index.html for human review and report.json for CI, automation, or AI clients.

What if the report is not applicable?

Check that the command ran from the actual application directory, then choose a more relevant pack. The selected pack here is jakarta-readiness.

Does it require a server?

No. Docker, GitHub Actions, static HTML, and JSON are enough for the first passive trial.
