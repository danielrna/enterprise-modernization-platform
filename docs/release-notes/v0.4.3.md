# v0.4.3

Benchmark validation timeout reliability for heavy Maven and Gradle repositories.

## Validation timeout cleanup

Harden benchmark validation timeout handling so stubborn child processes cannot leave benchmark publishing stuck after timeout evidence is captured.

Audience: Maintainers refreshing checkout-backed benchmark evidence on heavy Maven and Gradle repositories

- Escalates timed-out validation commands from `SIGTERM` to `SIGKILL` after a short grace period.
- Force-resolves timed-out validation commands so inherited Maven, Surefire, Gradle, or worker stdio cannot hang the benchmark publisher.
- Keeps timeout evidence explicit with failed validation checks, timeout output, duration, and exit code `124`.
- Adds a regression test with a fake Maven wrapper that leaves an attached child process ignoring `SIGTERM`.
- Preserves current evidence at 75 public reports, 55 checkout-backed reports, and 17 passing validation reports.
