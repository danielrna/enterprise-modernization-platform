# v0.4.1

Benchmark report-size hygiene for large checkout-backed evidence reports.

## Benchmark finding compaction

Cap detailed finding arrays in benchmark report JSON while preserving full totals, summaries, examples, next actions, and validation evidence.

Audience: Consultants and readers opening large public benchmark JSON reports

- Adds compact benchmark report JSON with `findingDetails` totals, production/test totals, detailed sample count, omitted count, and compaction policy.
- Keeps `findingSummary` as the authoritative full-count summary for finding types, top modules, and examples.
- Updates benchmark publishing, Migration Hub summaries, and MCP benchmark summaries to use full finding totals when reports are compacted.
- Regenerates oversized published benchmark reports, reducing Baeldung Hibernate Tutorials from about 12 MB to about 260 KB while preserving 17,621 total findings.
- Preserves the `v0.4.0` evidence counts: 75 public reports, 50 checkout-backed reports, and 17 passing validation reports.
