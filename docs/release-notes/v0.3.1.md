# v0.3.1

GitHub Pages deployment recovery for the static evidence site.

## Pages workflow recovery

Restore explicit GitHub Pages deployment after branch-only publishing left the public site on an old snapshot.

Audience: Consultants and users opening public benchmark, release-note, and Migration Hub URLs

- Restores `actions/configure-pages@v5`, `actions/upload-pages-artifact@v3`, and `actions/deploy-pages@v4` in `.github/workflows/pages.yml`.
- Uses unique Pages artifact names per workflow run attempt to avoid duplicate artifact collisions on reruns.
- Restores the `github-pages` deployment environment and deployment URL output.
- Verifies the `v0.3.0` release notes, Spring Cloud benchmark reports, and Spring Boot evidence hub return `200` from GitHub Pages.
- Preserves the `v0.3.0` evidence counts: 75 public reports, 45 checkout-backed reports, and 16 passing validation reports.
